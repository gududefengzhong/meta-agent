# ANDREJ_KARPATHY_PRINCIPLES.md - 开发原则附录

本文件提供通用开发原则，用于指导实现风格和决策取向。
如果与仓库协作规则或产品规格冲突，以 `CLAUDE.md` 和 `AGENT_SPEC.md` 为准。

## Think Before Coding
解决问题：错误假设、认知模糊、缺失取舍
- 先明确问题边界、前提假设和验收标准，再进入实现
- 先识别隐藏依赖、约束条件和失败模式，再决定方案
- 先比较方案，再选择路径，不用“先写出来再说”代替思考

## Simplicity First
解决问题：过度设计、提前抽象、不必要复杂度
- 优先最简可用实现
- 没有真实需求前，不做泛化和过早抽象
- 能直接表达的逻辑，不额外增加层级和包装

## Surgical Changes
解决问题：越界修改、改动扩散、破坏正交性
- 改动保持局部、精准、可解释
- 只修改完成当前目标必须触及的部分
- 不顺手重构无关模块，不把当前任务变成全面清理

## Goal-Driven Execution
解决问题：盲目开发、没有验收、无法证明完成
- 先定义成功标准，再落地实现
- 优先让验证方式和实现一起收敛
- 每一步改动都应能回答“这是否让目标更接近完成”
